## Short English post

```text
I’m building PromptBridge Arabic, an open-source tool for Arabic-speaking developers who write coding prompts in Arabic or Egyptian Arabic.

It converts Arabic developer intent into structured English prompts for AI coding tools.

Current workflows:
- CLI
- Browser extension
- VS Code-compatible extension
- Clipboard/Raycast shortcuts
- Secret redaction
- No API key, no external AI calls

Looking for feedback from Arabic-speaking developers using ChatGPT, Codex, Cursor, Claude, Gemini, or other coding agents.

https://github.com/ZiadEldesoky/promptbridge-arabic
```

## Thread outline

```text
1/ I’m building PromptBridge Arabic for Arabic-speaking developers using AI coding tools.

2/ The problem: many developers think through bugs, UI details, and product behavior naturally in Arabic, but coding agents usually work better with structured English prompts.

3/ PromptBridge converts Arabic/Egyptian Arabic developer intent into structured English prompts while preserving code, paths, commands, stack traces, packages, URLs, and env vars.

4/ It currently supports CLI, browser extension, VS Code-compatible extension, clipboard workflows, Raycast helper, modes, bilingual output, and optional secret redaction.

5/ It is local-first: no API key required and no external AI calls.

6/ I’m looking for feedback from Arabic-speaking developers and AI coding tool users.

Repo: https://github.com/ZiadEldesoky/promptbridge-arabic
```
